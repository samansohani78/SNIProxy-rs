About HTTP/2

HTTP/2 is a major revision of the HTTP network protocol used on the World Wide Web. It was derived from the earlier experimental SPDY protocol, originally developed by Google.

Binary Protocol

HTTP/2 uses binary framing instead of the text-based approach of HTTP/1.x, making it more efficient to parse and less error-prone.

Multiplexing

HTTP/2 can send multiple requests and responses in parallel over a single TCP connection, eliminating the need for multiple connections.

Server Push

HTTP/2 allows the server to proactively send resources to the client that it anticipates the client will need.

Header Compression

HTTP/2 uses HPACK compression to reduce overhead, particularly with redundant headers in multiple requests.

Stream Prioritization

HTTP/2 allows clients to specify which resources are more important, helping optimize delivery of critical resources.
